# Мониторинг на базе Prometheus, Alertmanager и Dex

## Описание проекта

Данный проект представляет собой готовый стек мониторинга для отслеживания состояния хоста и Docker-контейнеров с использованием **Prometheus**, **Alertmanager**, **Node Exporter** и **Dex** (Docker-экспортер). Решение предназначено для быстрого развертывания системы мониторинга с алертами и уведомлениями по электронной почте через Yandex.

Система автоматически отслеживает ключевые метрики хоста (память, диск, CPU) и Docker-контейнеров, отправляя оповещения при превышении критических порогов.

---

## Состав стека

| Сервис | Назначение | Порт |
|--------|------------|------|
| **Prometheus** | Сбор и хранение метрик, обработка правил алертов | `9090` |
| **Alertmanager** | Обработка алертов и отправка уведомлений | `9093` |
| **Node Exporter** | Сбор метрик хоста (CPU, память, диск, сеть) | `9100` |
| **Dex** | Docker-экспортер для сбора метрик контейнеров | `8386` |

Все сервисы работают в изолированной сети `monitoring` и общаются друг с другом по именам контейнеров.

---

## Требования к системе

- **Docker** (версия 20.10+)
- **Docker Compose** (версия 2.0+)
- **Операционная система**: Linux (поддержка `/var/run/docker.sock`)
- **Свободные порты**: `9090`, `9093`, `8386`
- **Доступ к интернету** для отправки почтовых уведомлений через Yandex SMTP

## Установка и запуск

### 1. Клонирование проекта
```bash
git clone <repository-url>
cd <project-directory>
```

### 2. Настройка конфигурации
Отредактируйте файл `alertmanager.yml` и укажите свои данные для почтовых уведомлений:
```yaml
email_configs:
  - to: 'ваша_почта@yandex.ru'          # Получатель
    from: 'отправитель@yandex.ru'       # Отправитель (должен совпадать с auth_username)
    auth_username: 'отправитель@yandex.ru'
    auth_password: 'ПАРОЛЬ_ПРИЛОЖЕНИЯ'  # Сгенерируйте в настройках Яндекса
```

> **Важно**: В Yandex используйте **пароль приложения**, а не пароль от аккаунта! [Инструкция по генерации](#-настройка-уведомлений)

### 3. Создание директорий для данных
```bash
mkdir -p prometheus-data alertmanager-data
```

### 4. Запуск стека
```bash
docker-compose up -d
```

### 5. Проверка статуса
```bash
docker-compose ps
```

Все контейнеры должны быть в статусе `Up` или `Running`.

## Структура конфигурационных файлов

| Файл | Назначение |
|------|------------|
| `docker-compose.yml` | Определение и связывание сервисов, монтирование томов, сетевые настройки |
| `prometheus.yml` | Основная конфигурация Prometheus: настройка сборки метрик (scrape), правила алертов, адрес Alertmanager |
| `rules.yml` | Правила алертов: условия срабатывания, пороги, уровни серьезности, аннотации для уведомлений |
| `alertmanager.yml` | Конфигурация Alertmanager: настройка почтового получателя, группировка и интервалы уведомлений |

## Описание алертов

### Алерты на состояние хоста

| Название алерта | Метрика | Условие | Уровень |
|-----------------|---------|---------|---------|
| **CA_High_memory_utilization** | Свободная память | Доступно менее 10% от общей | `critical` |
| **CA_Disk_space_problem** | Свободное место на диске | Доступно менее 10% на `/` | `critical` |
| **CA_node_exporter_down** | `up{job="node"}` | Экспортер не отвечает | `critical` |
| **CA_High_CPU_utilization** | Load average за 1 минуту | > 0.9 | `warning` |

### Алерты на Docker-контейнеры

| Название алерта | Метрика | Условие | Уровень |
|-----------------|---------|---------|---------|
| **Docker_HighCPUUsage** | `dex_cpu_utilization_percent` | > 80% в течение 5 минут | `warning` |
| **Docker_HighMemoryUsage** | `dex_memory_utilization_percent` | > 85% в течение 5 минут | `warning` |
| **Docker_ContainerDown** | `dex_container_running` | = 0 (контейнер остановлен) | `critical` |
| **Docker_ContainerRestarting** | `rate(dex_container_restarts_total[5m])` | > 0.1 перезапусков/сек | `warning` |
| **Docker_ExporterDown** | `up{job="dex"}` | Экспортер не отвечает | `critical` |

## 📧 Настройка уведомлений

Для отправки уведомлений на почту Yandex выполните следующие шаги:

1. **Войдите** в аккаунт Yandex
2. **Перейдите** в [Настройки безопасности](https://id.yandex.ru/security)
3. **Создайте** пароль приложения:
   - Перейдите в раздел "Пароли приложений"
   - Нажмите "Создать новый пароль"
   - Выберите тип "Почта" и назначьте имя (например, "Monitoring")
   - Скопируйте сгенерированный пароль
4. **Вставьте** полученный пароль в `alertmanager.yml` вместо `ПАРОЛЬ_ПРИЛОЖЕНИЯ`
5. **Перезапустите** Alertmanager:
   ```bash
   docker-compose restart alertmanager
   ```

## Доступ к веб-интерфейсам

| Сервис | Адрес | Назначение |
|--------|-------|------------|
| **Prometheus** | [http://localhost:9090](http://localhost:9090) | Graph, выражений и статус таргетов |
| **Alertmanager** | [http://localhost:9093](http://localhost:9093) | Просмотр и управление алертами |
| **Dex** | [http://localhost:8386](http://localhost:8386) | Метрики Docker-контейнеров |

### Доступ через SSH-туннель (для удаленных серверов)

Если мониторинг развернут на удаленном сервере без публичного доступа к веб-интерфейсам, используйте **SSH-туннелирование** для безопасного доступа:

```bash
# Проброс порта Prometheus
ssh -L 127.0.0.1:9090:127.0.0.1:9090 user@your-server-ip

# Проброс порта Alertmanager
ssh -L 127.0.0.1:9093:127.0.0.1:9093 user@your-server-ip

# Проброс порта Dex
ssh -L 127.0.0.1:8386:127.0.0.1:8386 user@your-server-ip

# Или все порты одной командой
ssh -L 9090:localhost:9090 -L 9093:localhost:9093 -L 8386:localhost:8386 user@your-server-ip
```

После установки туннеля откройте в браузере на своей локальной машине:
- [http://localhost:9090](http://localhost:9090) — Prometheus
- [http://localhost:9093](http://localhost:9093) — Alertmanager
- [http://localhost:8386](http://localhost:8386) — Dex

> **Советики**: Для постоянного использования добавьте параметр `-Nf` для фонового режима:
> ```bash
> ssh -Nf -L 9090:localhost:9090 -L 9093:localhost:9093 -L 8386:localhost:8386 user@your-server-ip
> ```

## Управление данными

Проект использует **persistent volumes** для сохранения данных между перезапусками:

| Директория | Назначение |
|------------|------------|
| `./prometheus-data/` | Хранение временных рядов Prometheus (TSDB) |
| `./alertmanager-data/` | Хранение состояния алертов и журнала уведомлений |

⚠️ **Важненько**: При удалении этих директорий данные будут потеряны. Рекомендуется настроить резервное копирование для production-сред.

## Возможные проблемы и их решение

### Ошибка: `cannot connect to alertmanager`
**Причина**: Prometheus не может достучаться до Alertmanager  
**Решение**: Проверьте, что все сервисы запущены:
```bash
docker-compose ps
```

### Ошибка: `dex container not found`
**Причина**: Dex не видит контейнеры  
**Решение**: Убедитесь, что монтирование `/var/run/docker.sock` работает корректно:
```bash
docker-compose logs dex | grep -i error
```

### Уведомления не приходят на почту
**Причина**: Неправильные настройки SMTP  
**Решение**:
- Проверьте правильность пароля приложения
- Убедитесь, что адрес `from` совпадает с `auth_username`
- Проверьте логи Alertmanager:
  ```bash
  docker-compose logs alertmanager | grep -i error
  ```

### Ошибка доступа к портам
**Причина**: Порты уже заняты другими сервисами  
**Решение**: Измените порты в `docker-compose.yml` и не забудьте обновить правила брандмауэра

### Метрики не собираются
**Причина**: Проблемы с сетью между сервисами  
**Решение**: Проверьте, что все сервисы в сети `monitoring`:
```bash
docker network inspect monitoring
```

## Заключение

Данный стек мониторинга готов к использованию и требует минимальной настройки. Он позволяет оперативно отслеживать состояние системы и контейнеров, получая уведомления о проблемах на электронную почту.

### Полезные команды

```bash
# Просмотр логов всех сервисов
docker-compose logs -f

# Перезапуск конкретного сервиса
docker-compose restart prometheus

# Остановка всех сервисов
docker-compose down

# Остановка с удалением томов (внимание: данные будут потеряны!)
docker-compose down -v
```
